Use the CLI stack when you want automation without the desktop app.

## Main commands

- `openwrk`: headless orchestrator (OpenCode + MAYA server + router)
- `maya-server`: filesystem-backed remote API server
- `opencode-router`: Slack/Telegram bridge with directory routing

## openwrk (recommended)

```bash
npm install -g openwrk
openwrk start --workspace /path/to/workspace --approval auto
```

Useful follow-ups:

```bash
openwrk status --maya-url http://127.0.0.1:8787 --opencode-url http://127.0.0.1:4096
openwrk approvals list --maya-url http://127.0.0.1:8787 --host-token <token>
```

## maya-server

```bash
npm install -g maya-server
maya-server --workspace /path/to/workspace --approval auto
```

Key env vars:

- `OPENWORK_TOKEN`
- `OPENWORK_HOST_TOKEN`
- `OPENWORK_APPROVAL_MODE`

## opencode-router

```bash
npm install -g opencode-router
opencode-router start
```

Pair it with bindings for deterministic routing by peer + directory.
